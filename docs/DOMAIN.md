# 도메인 모델 & 상태머신

## 핵심 도메인 객체

### User (고객)
- 회원가입/로그인
- 프로필 관리
- 주소록 (자주 가는 곳)
- 마일리지 보유
- 카드 등록 (최대 3장)
- 추천인 코드

### Driver (기사)
- 회원가입 (관리자 승인 필요)
- 면허증 / 차량 등록증 서류 제출
- 현재 위치 (실시간)
- 수락/거절 통계
- 누적 수입 / 정산 내역

### Ride (운행) ← 가장 복잡한 도메인
- 상태머신으로 관리 (아래 참조)
- 출발지 / 목적지 (위도/경도 + 주소 텍스트)
- 예상 거리 / 예상 시간
- 실제 요금
- 호출 타입 (프리미엄 / 빠른 / 일반)
- 결제 방식 (현금 / 포인트 / 카드)

### Payment (결제)
- 운행 1건 = 결제 1건
- PG사 트랜잭션 ID 저장
- 상태: pending → paid / failed / refunded

### Settlement (정산)
- 기사별 일 단위 정산
- 총 운임 - 수수료(%) = 기사 수령액

## 운행(Ride) 상태머신

### 상태 목록

```
PENDING       — 고객이 호출 완료, 배차 대기 중
MATCHED       — 기사 배정됨, 기사 수락 대기 중
ACCEPTED      — 기사가 수락함, 출발지로 이동 중
ARRIVED       — 기사가 출발지 도착
DRIVING       — 운행 중 (탑승 후 목적지 이동)
COMPLETED     — 운행 완료, 결제 완료
CANCELLED     — 취소됨 (고객/기사/시스템 중 하나가 취소)
FAILED        — 배차 실패 (주변 기사 없음, 타임아웃)
```

### 상태 전이 다이어그램

```
                    고객이 호출
                        │
                        ▼
                    PENDING ──────────────────────────────► FAILED
                        │                (타임아웃 or 주변 기사 없음)
                  배차 알고리즘
                        │
                        ▼
                    MATCHED ──────────────────────────────► CANCELLED
                        │          (기사 거절 or 타임아웃)   (시스템)
                  기사 수락
                        │
                        ▼
                   ACCEPTED ─────────────────────────────► CANCELLED
                        │              (고객 or 기사 취소)
                  기사 도착
                        │
                        ▼
                   ARRIVED ──────────────────────────────► CANCELLED
                        │              (고객 or 기사 취소)
                  고객 탑승 확인
                        │
                        ▼
                   DRIVING ──────────────────────────────► CANCELLED
                        │                    (기사 취소, 예외)
                  목적지 도착 + 결제
                        │
                        ▼
                   COMPLETED
```

### 상태 전이 규칙 (누가 언제 변경할 수 있는가)

| 현재 상태 | 다음 상태 | 트리거 | 비고 |
|----------|----------|--------|------|
| - | `PENDING` | 고객이 POST /rides | |
| `PENDING` | `MATCHED` | 시스템 (배차 알고리즘) | 자동 |
| `PENDING` | `FAILED` | 시스템 | 5분 내 배차 불가 |
| `MATCHED` | `ACCEPTED` | 기사 앱 (수락 버튼) | |
| `MATCHED` | `CANCELLED` | 시스템 | 기사 30초 내 미응답 → 다음 기사 재배차 또는 실패 |
| `ACCEPTED` | `ARRIVED` | 기사 앱 (도착 버튼) | |
| `ACCEPTED` | `CANCELLED` | 고객 앱 or 기사 앱 | 취소 수수료 정책 적용 |
| `ARRIVED` | `DRIVING` | 기사 앱 (출발 버튼) | |
| `ARRIVED` | `CANCELLED` | 고객 앱 or 기사 앱 | 취소 수수료 정책 적용 |
| `DRIVING` | `COMPLETED` | 기사 앱 (완료 버튼) | 결제 자동 처리 |
| `DRIVING` | `CANCELLED` | 관리자만 가능 | 극히 예외적 상황 |

## 배차 알고리즘 (matchingService)

### 기본 로직

```
1. 고객 출발지 기준 반경 N km 이내 ONLINE 상태 기사 조회
2. 거리 오름차순 정렬
3. 1순위 기사에게 콜 요청 전송 (Socket 이벤트)
4. 30초 대기
   - 수락 → MATCHED → ACCEPTED
   - 거절 or 타임아웃 → 다음 기사에게 재시도
5. 모든 기사 거절 or 주변 기사 없음 → FAILED
```

### 호출 타입별 매칭 조건

| 타입 | 설명 | 매칭 기사 조건 |
|------|------|---------------|
| `PREMIUM` | 프리미엄 | 프리미엄 등록 기사 |
| `QUICK` | 빠른 | 반경 3km 이내 + 최근접 |
| `NORMAL` | 일반 | 반경 5km 이내 + 최근접 |

### 기사 상태 (Driver.status)

```
OFFLINE   — 오프라인 (앱 종료 or 로그아웃)
ONLINE    — 대기 중 (콜 수신 가능)
BUSY      — 운행 중 (콜 수신 불가)
```

## 취소 정책

### 취소 주체별 수수료

| 취소 시점 | 고객 취소 | 기사 취소 |
|----------|----------|----------|
| PENDING → 취소 | 무료 | - |
| ACCEPTED (기사 이동 중) | 무료 (3분 이내) | 패널티 1회 |
| ACCEPTED (3분 초과) | 취소 수수료 3,000원 | 패널티 1회 |
| ARRIVED 이후 | 취소 수수료 5,000원 | 패널티 1회 |
| DRIVING 중 | 관리자만 취소 가능 | 관리자만 취소 가능 |

### 기사 패널티 규칙

```
패널티 누적 1회  → 경고 (알림)
패널티 누적 3회  → 24시간 콜 수신 정지
패널티 누적 7회  → 관리자 검토 후 계정 정지
```

## 결제 흐름

```
운행 완료 (DRIVING → COMPLETED)
    │
    ▼
결제 방식 확인
    ├─ 카드: POST /payments/approve (PortOne API 호출)
    ├─ 포인트: 마일리지 차감 (DB 트랜잭션)
    └─ 현금: 결제 레코드만 생성 (paid=true, method=CASH)
    │
    ▼
결제 성공 → Ride.status = COMPLETED
결제 실패 → Payment.status = FAILED
           → 재처리 큐에 등록 (Redis)
           → 3회 재시도 후 실패 시 관리자 알림
```

## 마일리지 (포인트) 규칙

| 발생 조건 | 지급량 |
|---------|--------|
| 운행 완료 | 운임의 1% |
| 추천인 코드로 가입 | 추천인에게 3,000원 지급 |
| 이벤트 지급 | 관리자 수동 지급 |
| 포인트 결제 | 차감 |

- 포인트 유효기간: 적립일로부터 1년
- 최소 사용 단위: 1,000원

## 정산 (Settlement)

```
매일 자정 배치 실행
    │
    ▼
전날 COMPLETED 운행 집계 (기사별)
    │
    ▼
총 운임 합계 - 플랫폼 수수료(20%) = 기사 수령액
    │
    ▼
Settlement 레코드 생성
    │
    ▼
기사 앱에서 정산 내역 조회 가능
```

수수료율 기본값: 20% (관리자 설정으로 변경 가능)
